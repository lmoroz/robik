# Code Review — 2025-07-14

## REVIEW SUMMARY

### Issue 1
- **Type:** Bug
- **Severity:** Critical
- **Location:** `Dockerfile:15`
- **Description:** [SYSTEM_PROMPT.md](SYSTEM_PROMPT.md) не копируется в Docker-образ. В `env.ts:31` файл ищется через `resolve` относительно `cwd` (`/app`), но Dockerfile копирует только [dist/](dist/) и [package.json](package.json). При `SYSTEM_PROMPT=YES` в [.env](.env) бот молча пропустит системный промпт, и «Робик»-персона не будет работать в Docker. Нужно добавить `COPY SYSTEM_PROMPT.md ./` в финальный stage.

### Issue 2
- **Type:** Bug
- **Severity:** Warning
- **Location:** [src/config/env.ts:49](src/config/env.ts:49)
- **Description:** `Number` не валидируется. Если пользователь задаст нечисловое значение (например, `OLLAMA_TIMEOUT_MS=abc`), результат будет `NaN`, и `AbortSignal.timeout` приведёт к непредсказуемому поведению. Рекомендуется добавить проверку: `const ms = Number(...); if (!Number.isFinite(ms) || ms <= 0) throw new Error(...)`.

### Issue 3
- **Type:** Bug
- **Severity:** Warning
- **Location:** [src/app.ts:17-19](src/app.ts:17-19)
- **Description:** `bot.start` возвращает `Promise`, который не обрабатывается (IDE: `Promise returned from start is ignored`). Если polling не запустится (невалидный токен), ошибка будет поймана только через `unhandledRejection`. Рекомендуется: `bot.start({...}).catch((err) => { console.error('Failed to start bot:', err); process.exit(1); });`.
